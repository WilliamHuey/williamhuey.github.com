Sprites and Link Padding

Sprites are often used to stylize ‘a’ tags. When a sprite sheet is used, a fixed width and height is used to specific the image on the sprite.

Here is a common example of how a sprite is commonly used

In this example the referenced image on the sprite is sufficiently large to allow for easy clicking. There are however times where this is not the case.

In other cases, padding is needed for the sprite image link to promote a greater clicking region for accessiblity. To achieve this, the markup must be changed such that the ‘a’ tag will have a ‘span’ and the id from the ‘a’ tag is placed on the ‘span tag’.

The span carries the image and the ‘a’ tag is now an expandable wrapper of ‘clickability’.

jQuery Event Handler Adapter

Bridging the Gap

Old version of jQuery are still being used (<= 1.6) and those versions do not support the new “on” method of attaching event handlers to elements. If you were authoring jQuery plugins and want to ensure compatibility when working with events then you would want to write an adapter inside your plugin.

The adapter works by performing a check to see if the “on” method is supported. If it is supported, then it is used, otherwise the older “bind” method is used instead. However, regardless of which method is supported both “on” and “bind” will be alias. The alias acts as a wrapper and delegates to which method is supported.

The alias “evtOn” and “evtOff” will be used for attaching and detaching event handlers.
